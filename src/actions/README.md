# 🚀 Actions Layer (Application Service)

이 폴더는 사용자의 요청이 처음 들어오는 **진입점**이자, 비즈니스 흐름을 관리하는 **오케스트레이터(Orchestrator)**입니다.

### 📜 지켜야 할 규칙
1.  **조립만 하세요**: 데이터를 직접 쿼리하거나 AI를 호출하지 말고, `repositories`를 불러서 결과를 조합하세요.
2.  **판단은 도메인에게**: "A면 B다" 같은 비즈니스 로직은 `domain` 레이어의 함수를 호출하여 결정하세요.
3.  **UI에 친화적으로**: 프론트엔드 컴포넌트가 바로 쓸 수 있도록 데이터를 예쁘게 가공해서 반환하세요.
4.  **인프라 금지**: `infrastructure` 폴더의 파일을 직접 `import` 하지 마세요. 반드시 `repositories`를 거쳐야 합니다.
